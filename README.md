# 📊 Inventory Monitoring & Optimization using SQL & Power BI

## 🚀 Project Overview
This project simulates a real-world retail analytics problem where inventory inefficiencies across multiple stores are analyzed and optimized using SQL.

---
## 📊 Dashboard Preview
![Dashboard](https://github.com/RageKrish/Inventory_Monitoring_-_Optimization_using_SQL_-_Power-BI/blob/main/PowerBI%20Dashboard/Screenshot.png)

---
## 🎯 Objectives
- Monitor stock levels across stores  
- Detect low inventory and stockout risks  
- Estimate reorder points using demand trends  
- Analyze inventory turnover and product performance  

---

## 🛠 Tech Stack
- SQL (MySQL)  
- Power BI  
- Excel / CSV  

---

## 📂 Project Structure
- `sql/` → All SQL scripts (schema + queries)  
- `dashboard/` → Power BI dashboard + screenshots  
- `docs/` → Documentation + ERD + summary  
- `outputs/` → KPI outputs (CSV files)  

---

## 📈 Key Features
- Low Inventory Detection  
- Days of Inventory Left  
- Inventory Turnover Analysis  
- Demand Decline Detection  
- Best-Selling Products by Region  

---

## 💡 Business Impact
- Identifies stockout risks early  
- Enables data-driven restocking decisions  
- Helps reduce overstock and holding cost  
- Projected improvement in restocking efficiency  

---

## 🧠 Learnings
- Writing scalable SQL queries  
- Translating business problems into KPIs  
- Building dashboards for decision-making  

---

## 📌 How to Use
1. Run SQL scripts from the `sql/` folder to create schema and generate KPIs  
2. Load output CSVs into Power BI  
3. Open `.pbix` file to view dashboard  

---

## ⭐ Project Highlight
This project demonstrates end-to-end data analysis — from raw data to business insights and visualization using SQL and Power BI.
